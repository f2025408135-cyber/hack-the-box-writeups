# Hack The Box: Cap

**Difficulty:** Easy
**OS:** Linux

Cap is an Easy-level Linux machine designed to demonstrate the risks of Insecure Direct Object Reference (IDOR) vulnerabilities and the danger of overly permissive Linux capabilities. The initial foothold involves identifying an IDOR flaw in a web application to download sensitive packet captures (PCAP files), which contain cleartext credentials. Privilege escalation is achieved by exploiting a Python binary assigned the `cap_setuid` capability.

---

## Reconnaissance

The assessment begins with an Nmap scan to enumerate open ports and running services.

```bash
nmap -p- --min-rate 10000 10.10.10.245
nmap -p 21,22,80 -sCV 10.10.10.245
```

The scan reveals three open ports:
*   **Port 21:** FTP (vsftpd 3.0.3)
*   **Port 22:** SSH (OpenSSH 8.2p1 Ubuntu)
*   **Port 80:** HTTP (gunicorn)

Anonymous FTP access is tested but disabled. We proceed to investigate the HTTP service running on port 80.

## Web Application Analysis

Navigating to port 80 reveals a web dashboard providing system network metrics. The application is running on Gunicorn, indicating a Python backend (likely Flask or Django).

Interacting with the web application, we identify a feature that allows users to capture and download network traffic (PCAP files) for analysis. When capturing traffic, the application redirects to a URL containing a numeric identifier, such as `http://10.10.10.245/data/3`.

### Exploiting IDOR for Initial Access

The numeric identifier in the URL (`/data/3`) is predictable. This pattern immediately suggests a potential Insecure Direct Object Reference (IDOR) vulnerability. An IDOR occurs when an application provides direct access to objects based on user-supplied input without properly verifying authorization.

We can manually modify the numeric ID in the URL to access previous captures. For example, changing the URL to `http://10.10.10.245/data/0` allows us to download the very first PCAP file generated by the system, which was created before our session began.

Tools like `ffuf` or Burp Suite Intruder can be used to rapidly iterate through IDs (e.g., 0 through 10) to identify and download all available captures.

```bash
ffuf -u http://10.10.10.245/data/FUZZ -w /usr/share/seclists/Discovery/Web-Content/burp-parameter-names.txt -mc 200
```

### PCAP Analysis

We download the identified PCAP files, particularly `0.pcap`, and open them in Wireshark for analysis.

Filtering the traffic in Wireshark (e.g., using the filter `ftp` or `tcp.port == 21`) reveals an unencrypted FTP session. By following the TCP stream of the FTP communication, we observe an authentication attempt in cleartext.

```text
USER nathan
PASS Buck3tH4TF0RM3!
```

The PCAP analysis yields valid credentials for the user `nathan`.

## Initial Access

We use the recovered credentials (`nathan:Buck3tH4TF0RM3!`) to authenticate against the SSH service on port 22.

```bash
ssh nathan@10.10.10.245
# Authentication successful
```
We obtain a shell as the user `nathan` and can read the `user.txt` flag located in the home directory. (These credentials also work for the FTP service, but SSH provides interactive command execution).

## Privilege Escalation

Once authenticated as `nathan`, we begin enumerating the system for privilege escalation vectors. We can use automated scripts like `LinPEAS` or perform manual checks.

A critical area of enumeration on Linux systems is checking for binaries with unexpected or overly permissive capabilities (capabilities allow processes to bypass certain kernel restrictions without needing full root privileges).

```bash
getcap -r / 2>/dev/null
```

The output reveals an interesting configuration:

```text
/usr/bin/python3.8 = cap_setuid,cap_net_bind_service+eip
```

The `/usr/bin/python3.8` executable has the `cap_setuid` capability assigned to it. The `cap_setuid` capability allows a process to manipulate its own UID (User ID). This means the Python binary can arbitrarily change its UID to 0 (root).

### Abusing Linux Capabilities

We can leverage this capability to spawn a root shell. We execute the Python binary and use the `os` module to set our UID to 0, followed by spawning a system shell (`/bin/bash`).

```bash
/usr/bin/python3.8 -c 'import os; os.setuid(0); os.system("/bin/bash")'
```

The command successfully sets the process UID to root and drops us into a root shell.

```bash
# id
uid=0(root) gid=1001(nathan) groups=1001(nathan)
```

The machine is fully compromised, and the `root.txt` flag is obtained from the `/root` directory.
